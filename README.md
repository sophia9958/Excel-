# Excel 多工作表合并工具

一个用于办公场景的 Excel 多 sheet 合并网页工具。

这个项目来自一次真实的办公需求：  
一个 Excel 文件里有很多个 sheet，格式差不多，但最后需要把所有 sheet 的内容合并到一张总表里。手动复制粘贴不仅慢，还容易漏 sheet、重复表头或字段错位。

于是我和 AI 一起把这个需求做成了一个可直接使用的网页工具。

## 项目特点

- 纯前端网页工具
- 支持上传 Excel 文件
- 自动识别多个 sheet
- 可勾选需要合并的工作表
- 支持按字段名对齐
- 支持跳过全空行
- 支持跳过数据区重复表头
- 保留来源文件、来源 sheet、来源行号
- 合并后可预览
- 支持导出新的 Excel 文件
- 导出时显示进度和状态
- 提供处理摘要 sheet
- 提供需复核明细 sheet
- 页面内置版本自检和脚本自检提示

---

## 适用场景

适合：

- 一个 Excel 文件里有多个 sheet
- 多个 sheet 表头基本一致
- 需要把多个 sheet 合并成一张总表
- 经常需要处理类似报表
- 希望同事也能直接使用，而不是每次都重新写脚本

不太适合：

- 每个 sheet 结构差异很大
- 有大量合并单元格
- 需要保留复杂公式、宏、图表
- 需要严格保留原始 Excel 样式
- 需要权限管理、多人协作、历史记录等复杂功能

## 功能说明

### 1. 上传 Excel

支持选择或拖拽上传 Excel 文件。

当前版本主要面向 `.xlsx` 文件。

### 2. 识别 sheet

上传后，工具会自动读取文件中的工作表，并显示：

- 文件名
- 工作表名
- 行数
- 列数
- 表头预览
- 是否可合并

### 3. 字段对齐方式

工具提供两种字段对齐方式：

#### 按字段名对齐，推荐

看表头名字合并。

例如：

| 批次 | 版块 | 片名 |
|---|---|---|

和：

| 片名 | 批次 | 版块 |
|---|---|---|

虽然列顺序不同，但只要表头名字能对上，就会合并到正确字段。

#### 按列位置合并

不看表头，只看第几列。

适合所有 sheet 列顺序完全一致的情况。

如果不确定，建议使用：

> 按字段名对齐

## 默认处理规则

### 跳过全空行

一整行都没有内容时，不放入总表。

### 跳过数据区重复表头

如果数据中间又出现一行表头，例如：

| 批次 | 版块 | 片名 |
|---|---|---|

工具会识别为重复表头，不当作数据行。

### 保留来源信息

合并结果中会保留：

- 来源文件
- 来源工作表
- 来源行号

这样后续发现问题时，可以快速回到原始 Excel 核对。

### 需复核提示

有些内容不一定是错的，但需要人工看一眼。比如：

- 字段为空
- 疑似说明行
- 疑似占位行
- 表头不一致
- 重复表头

工具不会直接删除这些不确定内容，而是标记为“需复核”。

## 导出内容

导出的 Excel 默认包含：

### 1. 合并结果

所有选中 sheet 合并后的总表。

### 2. 处理摘要

用于核对每个 sheet 的处理情况，包括：

| 字段 | 说明 |
|---|---|
| 文件名 | 来自哪个 Excel 文件 |
| 工作表 | 来自哪个 sheet |
| 原始总行数 | 原 sheet 总共有多少行 |
| 原始非空行数 | 原 sheet 中有内容的行数 |
| 实际合并行数 | 最终进入总表的行数 |
| 跳过全空行 | 被跳过的空行数量 |
| 跳过重复表头 | 被跳过的重复表头数量 |
| 需复核行数 | 被标记为需复核的行数 |
| 表头匹配率 | 当前 sheet 与基准表头的匹配程度 |

### 3. 需复核明细

列出需要人工确认的位置，帮助排查问题。

## 使用方法

### 本地使用

1. 下载 `index.html`
2. 用 Chrome 或 Edge 打开
3. 上传 Excel 文件
4. 选择需要合并的 sheet
5. 点击“合并并预览”
6. 核对处理摘要
7. 点击“导出 Excel”
